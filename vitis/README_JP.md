## アプリケーション概要

このプロジェクトには6つの主要なアプリケーションが含まれており、それぞれがZynq-7000プラットフォームの特定の機能をテスト・デモンストレーションするように設計されています：

1. **LED外部テスト1** - 直接GPIOレジスタアクセスによるLED制御
2. **LED外部テスト2** - Xilinx GPIOドライバベースのLED制御
3. **lwIPエコーサーバー** - lwIPスタックを使用したネットワークエコーサーバー
4. **メモリテスト** - 包括的なメモリ検証
5. **ペリフェラルテスト** - 複数ペリフェラル機能テスト
6. **PS LCD** - PS GPIO経由のLCDディスプレイ制御

## ハードウェア要件

- Zynq-7000ベースの開発ボード
- 必要なペリフェラルを設定したFPGAビットストリーム
- LED（LEDテストアプリケーション用）
- イーサネット接続（lwIPエコーサーバー用）
- LCDディスプレイ（PS LCDアプリケーション用）
- プログラミング/デバッグ用のJTAGまたはUART接続
- 適切な電源供給

## ソフトウェア要件

- Xilinx Vitis IDE
- Xilinx Vivado Design Suite
- Zynq-7000 BSP（ボードサポートパッケージ）
- 必要なXilinxドライバライブラリ
- UART通信用端末アプリケーション

## はじめに

1. このリポジトリをクローンまたはダウンロード
2. Vitis IDEを開きワークスペースをインポート
3. 目的のアプリケーションプロジェクトをビルド
4. 適切なビットストリームでFPGAをプログラム
5. アプリケーションを実行しUART経由で出力を監視

## アプリケーション

### LED外部テスト1

**場所:** `led_ext_1/`

**概要:** 直接GPIOレジスタアクセスを使用して基本的なLED制御をデモンストレーションします。コンソール出力付きの8状態LED点滅パターンを作成します。

**主な機能:**
- GPIOレジスタへの直接メモリアクセス
- 8状態LEDパターン（0x0から0x7）
- コンソールデバッグ出力
- 連続動作

**技術詳細:**
- LED制御: `XPAR_GPIO_0_BASEADDR + 0x00`
- 方向制御: `XPAR_GPIO_0_BASEADDR + 0x04`
- 遅延: 40,000,000回の反復

**著者:** Copyright (C) 2020 Cobac.Net All Rights Reserved

---

### LED外部テスト2

**場所:** `led_ext_2/`

**概要:** 公式Xilinx GPIOドライバ（XGpio）を使用したLED制御。適切な初期化とエラーハンドリング付き。

**主な機能:**
- Xilinx GPIOドライバ実装
- 3LED制御（ビット0-2）
- エラーチェック付きドライバ初期化
- 設定可能なタイミング

**技術詳細:**
- GPIOチャネル: 1
- LED定義: LED1 (0x01), LED2 (0x02), LED3 (0x04)
- 遅延: 50,000,000回の反復
- デバイスID: `XPAR_GPIO_0_DEVICE_ID`

---

### lwIPエコーサーバー

**場所:** `lwip_echo_server/`

**概要:** 軽量IPスタック（lwIP）を使用したネットワークエコーサーバー実装。IPv4およびIPv6プロトコルをサポート。

**主な機能:**
- ポート7でのTCP/UDPエコーサーバー
- IPv4およびIPv6サポート
- 設定可能なネットワーク設定
- プラットフォーム固有の最適化

**ネットワーク設定:**
- **IPv4:** 192.168.1.10/255.255.255.0, ゲートウェイ: 192.168.1.1
- **IPv6:** FE80:0:0:0:20A:35FF:FE00:102
- **MAC:** 00:0a:35:00:01:02

**使用例:**
```bash
# IPv4テスト
telnet 192.168.1.10 7

# IPv6テスト
telnet -6 FE80:0:0:0:20A:35FF:FE00:102%eth1 7
```

**参考文献:** XilinxアプリケーションノートXAPP 1026

### メモリテスト

**場所:** `memory_test/`

**概要:** ハードウェア設計に存在するすべてのメモリ領域を検証する包括的なメモリテストアプリケーション。

**主な機能:**
- メモリ領域検出とテスト
- 複数のテストパターン
- キャッシュ対応テスト（D-Cache無効）
- 包括的なエラーレポート

**技術詳細:**
- 4KBチャンクでテスト
- Xilinxメモリテストライブラリ使用
- ヒープ割り当てなし（最小フットプリント）
- プラットフォーム固有の最適化

**ライセンス:** SPDX-License-Identifier: MIT

---

### ペリフェラルテスト

**場所:** `peripheral_test/`

**概要:** ポーリングモードと割り込み駆動モードの両方で、様々なZynq-7000ペリフェラルをテストする包括的なテストスイート。

**サポートされるペリフェラル:**
- GPIO, UART（ポーリング/割り込み）, I2C, SPI
- タイマー（ポーリング/割り込み）, ウォッチドッグ（割り込み）
- DMA（割り込み）, デバイス設定
- 割り込みコントローラ

**テストモジュール:**
- 各ペリフェラルの個別テストファイル
- 自己テスト機能
- エラー検出とレポート
- パフォーマンスメトリック

**主な機能:**
- スタンドアロンドライバライブラリ使用
- ARM Cortex-A9サポート
- リソース管理
- 包括的なエラーハンドリング

---

### PS LCD

**場所:** `ps_lcd/`

**概要:** EMIOピンを介してProcessing System GPIO機能を使用するLCD制御アプリケーション。

**主な機能:**
- PS GPIO経由のLCD制御
- EMIOピン設定
- カラーディスプレイサポート
- LCD初期化シーケンス

**EMIOピンマッピング:**
- チップセレクト: ピン54
- コマンド/データ: ピン55
- リセット: ピン56
- シリアルクロック: ピン57
- シリアルデータ: ピン58

**カラーサポート:**
- 16個の事前定義色（WHITE, BLACK, RED, GREEN, BLUEなど）
- RGBカラーレンダリング
- ディスプレイ管理

**ライセンス:** SPDX-License-Identifier: MIT

## 共通開発ワークフロー

1. **ハードウェアセットアップ**
   - VivadoでFPGA設計を設定
   - ビットストリームを生成しVitisにエクスポート
   - 必要なペリフェラルを接続

2. **ソフトウェア開発**
   - アプリケーションをVitisワークスペースにインポート
   - BSP設定を構成
   - アプリケーションをビルド

3. **テストとデバッグ**
   - ビットストリームでFPGAをプログラム
   - JTAGまたはメモリからアプリケーションを実行
   - デバッグのためにUART出力を監視

4. **検証**
   - ハードウェア機能を検証
   - エラーのコンソール出力を確認
   - アプリケーション固有の機能をテスト

## トラブルシューティング

### 一般的な問題

1. **ビルドエラー**
   - BSP設定を確認
   - ドライバライブラリの包含を確認
   - 正しいハードウェアプラットフォームを確認

2. **ランタイムエラー**
   - FPGAプログラミングを確認
   - ペリフェラル接続を確認
   - エラーメッセージのUARTを監視

3. **ハードウェア問題**
   - 電源供給を確認
   - クロック設定を確認
   - ピン割り当てを検証

### デバッグのヒント

- デバッグ情報にUART出力を使用
- ハードウェア設計がソフトウェアの期待と一致することを確認
- 割り込みコントローラ設定を確認
- メモリマップとアドレス空間を検証
